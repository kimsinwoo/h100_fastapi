# 개체 동일성 유지 파이프라인 (Identity-Preserving Pipeline)

프롬프트로는 해결되지 않는 이유와, 서비스 수준으로 안정화하기 위한 **“개체를 생성 모델의 재해석 대상에서 빼는”** 구조를 정리한 문서.

---

## 1. 왜 동일 개체가 유지되지 않는가 (기술적 원인)

### A. Turbo(증류/가속) 계열의 구조적 한계

- Turbo/LCM류는 **적은 스텝에서 빠르게 수렴**하도록 정답 분포를 거칠게 맞춤.
- **미세한 고유 특징**(눈 간격, 코/주둥이 비율, 털 결 패턴) 같은 reconstruction detail은 잘 보존하지 못함.
- **클래스/스타일 특징**(‘귀여운 하얀 강아지’) 쪽으로 빨려감 → identity 보존보다 “그럴듯함/스타일 일관성”에 최적화.

### B. strength(denoise) = “정체성 파괴량”

- img2img에서 strength는 **원본을 몇 %까지 버리고 다시 샘플링할지**.
- **전체 프레임에 노이즈**가 주입되면, 모델은 “원본 복원”이 아니라 **“조건을 만족하는 이미지를 다시 생성”**.
- 포즈/구도는 큰 구조라 남기 쉬우나, **개체 동일성은 고주파(미세 특징)**라 strength가 조금만 올라가도 쉽게 바뀜.
- 768 해상도 + 전체 img2img + Turbo 조합:
  - **0.35~0.45**만 넘어도 ‘다른 개체’로 재해석이 매우 흔함.
  - **0.5~0.6**은 “포즈만 비슷한 다른 강아지”가 나오는 게 정상에 가깝음.

### C. Stable Diffusion엔 “동일 개체” 개념이 없음

- 텍스트로 “same individual”을 넣어도 효과가 제한적인 이유:
  - SD에는 **‘이 픽셀의 이 강아지 = 그 강아지’라는 ID 토큰이 없음**.
  - 텍스트는 **분포를 밀어주는 약한 조건**일 뿐.
  - **강한 노이즈** 상황에서는 텍스트 조건이 ‘정확한 개체’가 아니라 **‘비슷한 개체’**로 만족되어도 손실이 없음.

**결론:** 개체 동일성 90~100%를 목표로 하면 **전체 img2img는 구조적으로 불리**하고, **inpaint/합성 기반**이 정답.

---

## 2. 전체 img2img vs inpaint: 구조적 차이

| 방식 | 동작 | 개체 |
|------|------|------|
| **전체 img2img** | 원본 전체 latent → 전체 노이즈 → 전체 디노이즈 | 개체도 배경도 **전부 재샘플링 대상** → “배경만 바꾸기”와 목표가 **구조적으로 충돌** |
| **inpaint(마스크)** | **바꿀 영역(배경)만** 디노이즈/재생성 | **안 바꿀 영역(반려동물)은 원본 latent/pixel 고정** → 동일 개체 유지를 파이프라인에서 **강제** 가능 |

---

## 3. 권장 서비스 아키텍처 (상용급)

### (0) 전처리: 반려동물 분리(세그멘테이션)

- **SAM2 / BiRefNet / U²-Net** 계열로 펫 마스크 생성.
- **털 가장자리**: 반드시 **feather(블러) + 약간 확장(dilate)** 해서 “털 halo” 포함.
  - 귀/털 끝이 잘리면 합성 티가 나고, 구름이 개체를 침범해 보임.
- **이 단계 품질이 전체 성공률의 70%를 좌우.**

### (1) 배경 생성: inpaint로 “배경만” 구름 생성

- **inpaint 모델 사용** (필수).
- 마스크: **배경=수정**, **반려동물=고정**.
- 여기서는 **strength를 높여도 됨** (개체 영역은 고정이므로).
- 구름처럼 큰 텍스처에는 **Turbo보다 SDXL Inpaint 계열**이 결과 안정성이 좋음.
- **ControlNet** (선택): Depth 또는 Canny를 **배경 inpaint 조건에만** 낮은 가중치로 사용. 전체에 강하게 걸면 개체 변형 가능성 있음.

### (2) 합성/하모나이징

- **2-A (추천)**: 픽셀 합성 + **톤매칭**
  - 원본 반려동물 픽셀 그대로 유지, 배경과 **색감/명암만** 맞추는 후처리 → 개체 동일성 **100%에 가깝게** 유지.
- **2-B (선택)**: **초저강도** 스타일 통일 패스
  - 전체를 한 번 더 img2img하되 **strength 0.05~0.12**, 반려동물 영역은 **마스크/latent 고정**.
  - 목적: 경계 자연화 + 분위기 통일. strength가 올라가면 identity 다시 흔들림.

### (3) 품질 게이트(QA): “동일 개체인지” 자동 판정 + 재시도

- **입력**에서 반려동물 crop → 임베딩 추출.
- **출력**에서 반려동물 crop → 임베딩 추출.
- **유사도가 임계치 미만**이면:
  - (i) 마스크 조정(확장/feather)
  - (ii) 배경 inpaint 파라미터 조정
  - (iii) 그래도 실패 시 **배경만 생성 + 원본 반려동물 100% 합성**으로 폴백.
- **CLIP** 유사도는 “같은 종/비슷한 강아지”에 관대 → **DINOv2류(자기지도 특징)**가 개체 특징 보존 판정에 유리. 실전에서는 CLIP+DINO 조합 게이트도 사용.

---

## 4. 모델 선택

| 목표 | 권장 |
|------|------|
| 동일 개체 90~100% + 배경만 완전 교체 | **전체 img2img는 구조적으로 불리.** 최소한 **inpaint로 배경만 수정** 구조로 변경. |
| Turbo 유지 | **배경 생성만** inpaint로 수행. (현재는 segment → solid에서 구름 생성 → 합성으로 유사 구조.) |
| 품질/안정성 상향 | 배경 생성은 **SDXL Inpaint(또는 고품질 XL 기반 inpaint)** 권장. Turbo는 “빠른 미리보기/저가 플랜” 용도로 한정. |

---

## 5. ControlNet / IP-Adapter / Identity embedding

- **ControlNet**: 구도/경계/공간감 안정화. “개체 동일성” 해결책은 아님. 잘못 쓰면 개체 변형 유발.
- **IP-Adapter(이미지 레퍼런스)**: inpaint와 함께 쓰면 개체 드리프트 감소에 도움. 완전 동일 보장은 아니며, **마스크 고정이 1순위**.
- **Identity embedding/게이트(재시도)**: 서비스 품질을 **통계적으로** 올리는 장치. “한 번에 완벽”보다 **실패 시 자동 재시도/폴백**이 체감 품질을 크게 올림.

---

## 6. 최종 권장 파이프라인 요약

1. **펫 세그멘테이션** (털 halo 포함: feather + dilate).
2. **SDXL Inpaint**로 **배경만** 생성 (volumetric clouds). 필요 시 Depth/Canny ControlNet **약하게** 보조.
3. **합성 + 톤매칭** 또는 **초저강도(0.05~0.12)** 통일 패스(펫 영역 고정).
4. **Identity QA**: 임베딩 유사도 게이트. 실패 시 마스크/파라미터 조정 재시도.
5. **최종 폴백**: “생성된 배경 + 원본 펫 100% 합성”.

이 구조에서는 **개체 동일성은 파이프라인이 강제**하고, **모델은 배경 생성에만** 집중하게 되어 “비슷한 다른 강아지 + 어정쩡한 배경” 문제가 **구조적으로** 사라짐.

## 7. 현재 코드와의 대응

| 단계 | 현재 구현 | 비고 |
|------|-----------|------|
| (0) 세그멘테이션 | `rembg` (U²-Net) | feather/dilate(털 halo) 미적용 → 추후 적용 권장 |
| (1) 배경만 생성 | solid 이미지에서 구름 img2img 후 **합성** (subject 미포함) | 진짜 inpaint(마스크 배경만)는 미구현. 합성으로 동일 효과 유사 달성 |
| (2) 합성 | `composite_foreground_on_background` | 톤매칭/초저강도 패스 미적용 |
| (3) QA 게이트 | 없음 | DINO/CLIP 유사도 + 재시도/폴백 추후 도입 권장 |
| Fallback | segment 실패 시 전체 img2img (strength 0.42, Turbo “수정” 모드) | identity 보장 안 됨, 최소한 완화 |

- **Cloud 성공 경로**: segment → 배경만 구름 생성 → 합성 → **img2img 없이 반환** → 개체는 diffusion 통과 안 함.
- **문서**: `app/services/background_replacement.py`, `app/services/image_service.py` 상단 주석에서 본 문서 참조.
